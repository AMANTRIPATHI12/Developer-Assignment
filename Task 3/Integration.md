# Task 3 – Integration Design

To build a reliable and scalable integration, I would use a **custom backend (Node.js/Express)** together with the **HubSpot CRM API**, **Karix WhatsApp Business API**, **Google Tag Manager (GTM)**, and **Google Ads Conversion Tracking**. I would avoid using Zapier or Make because this workflow is business-critical, requires better control over validation, error handling, logging, and future scalability.

The workflow begins when a user submits the consultation form on the landing page. The frontend validates the input and pushes the `consultation_form_submitted` event to the `dataLayer` for GTM and GA4 tracking. Simultaneously, it sends the form data to a secure backend API. The backend first validates the request and then searches HubSpot using the submitted phone number through the CRM Search API. Since HubSpot does not deduplicate contacts by phone by default, I would explicitly search for an existing contact with the same phone number. If a contact exists, I would update it; otherwise, I would create a new contact with the fields: Name, Phone, Clinic Preference, Source = "Google Ads - Consultation Landing Page", and Lead Status = "New Enquiry".

After HubSpot successfully creates or updates the contact, the backend triggers the Karix WhatsApp Business API to send a confirmation message to the patient. At the same time, GTM records the successful `consultation_form_submitted` conversion, which is imported into Google Ads for campaign optimization.

The biggest failure point is the CRM integration. If HubSpot is temporarily unavailable or returns an API error, lead data could be lost. To prevent this, I would implement a retry mechanism with exponential backoff and store failed requests in a database or message queue for automatic reprocessing. Every API request and response would also be logged to simplify troubleshooting.

To ensure the WhatsApp confirmation is sent within the required two-minute SLA, I would monitor API response times, queue length, delivery status, and failures using application logs and automated alerts. If message delivery fails after multiple retries, the system would notify the operations team immediately so the patient can be contacted manually. This approach ensures reliable lead capture, prevents duplicate contacts based on phone numbers, maintains high data quality, and provides continuous monitoring for production reliability.
